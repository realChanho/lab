# Create Note

`agentic-engineering` 노트들이 쓰는 통일된 에디토리얼 디자인으로 한국어 요약/기준 HTML을 만든다.

## 언제 쓰나

- 영상·아티클·자료를 한국어로 요약해 HTML 노트로 남길 때
- 프로젝트 기준 문서를 HTML로 정리할 때
- 기존 `agentic-engineering/*/summary.html`, `index.html`과 톤을 맞춰야 할 때

## 디자인 원칙 (반드시 지킬 것)

- **카드·둥근 박스·그라데이션·원형 번호 배지·이모지 금지.** AI 생성물 클리셰를 피한다.
- 본문 폭 `660px`, 글자 `17px`, 줄간격 `1.85`. 가독성 최우선.
- 제목·리드·결론은 **세리프(Georgia)**, 본문은 산세리프. 위계를 폰트로 가른다.
- 색은 다크 단색 + 골드 액센트(`--accent: #d6a35c`) 하나로 절제. 위험 표시만 `--danger`.
- 구획은 박스가 아니라 **얇은 가로 구분선(1px `--rule`)** 으로 나눈다.
- 단계 목록은 `01·02` leading-zero 번호. 정의형 항목은 `dt/dd` 2열 그리드(모바일 1열).
- 한국어 문장은 마침표로 끝낸다. 콜론으로 끝내지 않는다. 콜론 대신 `—`(em dash) 사용.
- 외부 자료 요약이면 출처(제목·링크·채널·길이·날짜)와 작성일을 `.source`에 명시.
- footer는 "이 노트는 ~ 요약·재구성한 것입니다." 식 한 줄. "자막 추출" 같은 표현은 쓰지 않는다.

## 생성 절차

1. 산출물 경로를 정한다. 주제별 폴더로 나누고 파일명은 항상 `index.html`로 한다: `agentic-engineering/<주제-슬러그>/index.html`.
2. 아래 `template.html`을 복사해 내용을 채운다. CSS는 그대로 두고 `<main>` 안만 교체한다.
3. 새 폴더면 두 곳에 등록한다 — `agentic-engineering/CLAUDE.md`의 "구조" 목록 한 줄, 그리고 `agentic-engineering/index.html` 목차의 `.toc`에 항목 한 개(`t-kind`·`t-title`·`t-desc`).
4. 파일을 브라우저 없이 점검한다. 모든 한국어 문장이 `.`/`?`/`!`로 끝나는지, 카드/그라데이션/이모지가 안 들어갔는지 확인한다.

## 컴포넌트 사용 기준

- `.lead` — 핵심 한 줄(세리프) + `.after`(부연, 산세리프 muted). 문서당 1개.
- `section` + `.num`(섹션 라벨) + `h2` + `.sub`(한 줄 설명) — 주요 구획 단위.
- `ol.steps` — 순서/단계/원칙. `.st`(제목), `.sd`(요약), 선택적 `ul`(세부).
- `dl.defs` > `.def`(dt/dd) — 용어·항목·표 대체. 비교/매핑에 적합.
- `pre`/`code` — 디렉터리 트리, 명령어. `code`는 인라인 골드.
- `table` — 행 구분선만, 헤더는 대문자 트래킹. 박스 테두리 없음.
- `.takeaway` — 결론(세리프) + `.after`. 문서 끝 1개.
- `.warn` / `.danger` — 주의·위험. 남용 금지.

전체 마크업·CSS 레퍼런스는 이 폴더의 `template.html`을 그대로 따른다.
