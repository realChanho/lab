CLAUDE.md 자동 관리 플러그인

영상 내용 한국어 요약 · 플러그인 공식 문서로 보강 · 작성일 2026-05-19

CLAUDE.md는 프로젝트가 진행될수록 비대해지고 낡는다. 공식 claude-md-management 플러그인은 세션에서 배운 것을 기록하고, 파일 자체를 정기 점검해 깨끗하게 유지한다. CLAUDE.md는 매 프롬프트마다 자동으로 따라붙는 프로젝트 규칙 파일이다. 늘어나는 규칙을 누가 정리할 것인가 — 이 플러그인이 그 자리를 맡는다.

01

왜 필요한가

CLAUDE.md가 비대해지는 구조적인 이유.

  1. 매번 따라붙는 파일
    CLAUDE.md는 클로드 코드가 알아야 할 규칙을 적는 파일이고, 프롬프트를 요청할 때마다 자동으로 컨텍스트에 실린다.
  2. 규칙은 계속 늘어난다
    프로젝트가 진행될수록 지켜야 할 규칙·관습·함정이 쌓이고, 파일이 자연스럽게 부풀어 오른다.
  3. 코드와 어긋난다
    명령어가 바뀌고 의존성이 갈리면서 CLAUDE.md만 옛 상태로 남는다. 누군가 주기적으로 점검해야 한다.
02

두 가지 도구

목적이 다르다. 하나는 기록, 하나는 검진.

/revise-claude-md
커맨드. 세션이 끝날 때 한 번 호출한다. 이번 대화를 분석해 새로 알게 된 명령어·작업 패턴·함정을 추출하고, CLAUDE.md에 추가할 후보를 보여 준다. 회의 끝에 쓰는 회의록과 같다.
claude-md-improver
스킬. 프로젝트의 모든 CLAUDE.md를 주기적으로 점검한다. 여섯 기준으로 평가·등급을 매기고, 작동하지 않는 명령어·빠진 의존성·누락된 환경 설정 같은 이슈를 짚어 준다.

트리거 — 전자는 세션이 끝났다는 시점에, 후자는 코드베이스가 바뀌었다는 시점에 작동한다.

03

언제 어떻게 부르나

두 도구의 호출 방식.

  1. 작업을 끝낸 직후
    세션을 닫기 전에 /revise-claude-md를 입력한다. 이번 대화에서 건진 것이 자동으로 정리돼 후보로 뜬다.
  2. 정기 점검
    "audit my CLAUDE.md files" 같은 한 줄로 claude-md-improver를 깨운다. 등급표와 수정 제안을 받은 뒤 승인하면 파일에 반영된다.
04

평가 여섯 기준

claude-md-improver가 점수를 매기는 축. 합계 100점.

기준배점확인 질문
Commands / Workflows20빌드·테스트·배포 명령어가 적혀 있는가.
Architecture Clarity20코드 구조를 이 파일만 보고 파악할 수 있는가.
Non-obvious Patterns15관습·함정·예외가 기록됐는가.
Conciseness15코드만 봐도 아는 내용을 늘어놓지 않는가.
Currency15현재 코드베이스 상태를 반영하는가.
Actionability15지시가 실행 가능한가, 모호하지 않은가.

등급은 A 90–100, B 70–89, C 50–69, D 30–49, F 0–29로 갈린다. 결과는 항상 보고서 형태로 먼저 보여 주고, 사용자 승인 후에야 파일을 고친다.

05

주로 잡아내는 이슈

점검에서 자주 발견되는 부패 유형.

Stale commands
예전엔 됐지만 지금은 작동하지 않는 빌드·테스트 명령어.
Missing dependencies
실행에 필요한데 문서에 적히지 않은 도구·패키지.
Outdated architecture
파일 구조가 바뀌었는데 옛 디렉터리 설명이 그대로 남은 경우.
Missing env setup
필요한 환경 변수·초기 설정이 문서에 없는 경우.
Undocumented gotchas
코드에 숨은 함정·관습이 어디에도 적혀 있지 않은 경우.
06

설치

공식 마켓플레이스의 플러그인이다.

claude plugin install claude-md-management@claude-plugins-official

제작자는 Anthropic의 Isabella He. 저장소는 anthropics/claude-plugins-official의 plugins/claude-md-management다.

CLAUDE.md를 코드처럼 관리한다 — 세션 끝엔 회의록, 주기적으론 건강검진.

기록 도구와 점검 도구를 분리한 것이 이 플러그인의 핵심이다. 둘 다 자동 적용 대신 후보·보고서를 먼저 보여 주고 승인받는다.
