How Claude Code Works

출처 How Claude Code Works · Claude (Anthropic 공식) · 2:50 · 업로드 2026-05-14
영상 내용 한국어 요약 정리 · 작성일 2026-05-17

Claude Code는 대화형 챗봇이 아니라, 스스로 도구를 호출하고 결과를 검증하며 반복하는 자율 에이전트다. 일반 챗 어시스턴트가 "텍스트 입력 → 텍스트 출력"이 전부인 것과 근본적으로 다르다. 이 차이를 만드는 네 가지 축을 정리한다.

01

에이전틱 루프

핵심 작동 방식. 다음 사이클을 목표가 검증될 때까지 반복한다.

  1. 프롬프트 입력
    사용자가 작업을 요청한다.
  2. 컨텍스트 수집
    작업에 필요한 정보를 모델과 상호작용하며 모은다. 모델은 텍스트 또는 tool call을 반환한다.
  3. 행동
    파일 수정, 명령어 실행 등 실제 작업을 수행한다.
  4. 검증
    결과가 원래 목표를 달성했는지 확인한다.
    • 달성했으면 — 종료하고 다음 프롬프트를 기다린다.
    • 못 했으면 — 완성·검증될 때까지 루프를 다시 돈다.

루프가 도는 동안 사용자는 언제든 컨텍스트 추가, 중단, 방향 수정으로 개입해 목표 쪽으로 유도할 수 있다.

02

관리되는 컨텍스트 윈도우

대화가 길어져도 흐름을 유지하는 메모리 관리 방식.

컨텍스트 윈도우
대화, 파일 내용, 명령 출력 등을 얼마나 저장하고 되돌아볼 수 있는지를 결정한다.
자동 압축
한계에 도달하면 무엇을 빼고 무엇을 요약할지 스스로 판단해 사용량을 다시 낮춘다.
03

도구

에이전트의 중추. "입력 → 출력" 사이에 행동을 끼워 넣는 장치.

역할
에이전트가 작업에 가까워지기 위해 언제 코드를 실행할지 스스로 결정하게 한다. 예: read file, search web.
호출 판단
Claude Code는 시맨틱 검색으로 어떤 도구를 호출하고 그 출력을 가져올지 판단한다.
04

설정 가능한 권한

Shift + Tab으로 세 모드를 전환한다.

기본
파일 수정·셸 명령 실행 전 명시적 허가를 요청한다.
Auto-accept edits
파일 편집은 묻지 않고 적용, 단 명령 실행은 여전히 확인한다.
Plan mode
읽기 전용 도구만 써서 작업 시작 전 실행 계획을 수립한다.

주의. 권한을 건너뛰면 Claude Code의 실수를 발생 전에 잡기 어려워진다. 명령 실행 전권을 줄 때는 신중해야 한다.

코드베이스를 읽고, 행동하고, 스스로 검증한다. 그 점이 챗 윈도우와 근본적으로 다르다.
